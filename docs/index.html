Project Overview

What We Built

Dark patterns are deceptive UX or marketing tactics that pressure users into choices they may not fully intend. This project first classifies short e-commerce text snippets as Dark Pattern or Not Dark Pattern, then estimates a likely type for suspicious text.

Research Question

How effectively can supervised text-classification models detect dark-pattern language, and what are the limits of a text-first consumer-protection tool?

Data and Pipeline

From Dataset to Prediction

Primary

Krish Uppal balanced dataset: 1,178 dark-pattern and 1,178 non-dark-pattern snippets.

Expanded

Devitachi and Akash Nath sources support robustness checks after cleaning and deduplication.

1. Normalize columns: text, label, category, source
2. Split data: 80/20 random stratified train/test split
3. Convert text to TF-IDF unigram + bigram features
4. Predict dark-pattern vs not-dark-pattern with Logistic Regression
5. Run a second-stage category model for likely type: urgency, scarcity, social proof, and more
Model

Why Logistic Regression?

It performed best on the primary balanced dataset, gives confidence scores, trains quickly, and stays interpretable through feature weights.

  • Type: supervised binary text classification
  • Input: website text snippet
  • Output: label + confidence score
  • Second stage: category classifier for suspicious snippets
Category Model

Likely Type Recognition

After the binary detector flags text as suspicious, a second model predicts the likely dark-pattern type. The best category model is Linear SVM with macro F1 around 0.894.

  • Common types: Scarcity, Social Proof, Urgency, Misdirection
  • Rarer types: Forced Action, Obstruction, Sneaking
  • Reported as a likely type because category labels are imbalanced

Two Ways to Try It

  • Safeguards: optional filters suppress plain discounts, product specs, testimonials, counters, and vague snippets unless pressure language is present.
Error Analysis

What We Learned From Testing

Real webpage scans initially flagged ordinary sale labels and catalog titles such as Previous price: $99.00 47% off or camera specs with US Stock.

The demo now filters these low-context false positives while still reviewing urgency/scarcity language like limited time, low stock, and only 2 left. Users can turn filters off to inspect raw model behavior.

Training Improvements

Without New Datasets

  • Hard-negative mining from existing scans
  • TF-IDF parameter tuning
  • Logistic Regression regularization tuning
  • Threshold tuning for precision/recall tradeoff
  • Cross-validation for stability
Responsible AI

Impact, Bias, and Production Limits

Positive Impact

Helps consumers notice manipulative language, supports website audits, and makes dark-pattern examples easier to discuss.

Risks

False positives can unfairly flag legitimate businesses; false negatives can miss real harms. Labels reflect human judgment and English e-commerce bias.

Limits

The project is text-first. Real production detection would need OCR, screenshot analysis, DOM/CSS features, color/layout signals, and scripted UX-flow testing.

Future Work

Next Steps

  • Use hard-negative mining to improve model training from current mistakes.
  • Try calibrated Linear SVM and transformer-based NLP models.
  • Add screenshot + OCR experiments for image-based text.
  • Evaluate UI flow manipulation in checkout, cancellation, cookie consent, and unsubscribe journeys.
  • Build a browser-extension prototype after stronger validation.
